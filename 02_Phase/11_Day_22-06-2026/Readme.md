## 1. Machine Learning Fundamentals

### What is Machine Learning?
Machine Learning (ML) is a subfield of Artificial Intelligence (AI) where computers study patterns in historical data to make decisions or predictions on new, unseen data, without being explicitly programmed.

```text
Artificial Intelligence (AI)
    └── Machine Learning (ML)
            └── Deep Learning (DL)
```

- **AI**: The broad concept of machines replicating human intelligence.
- **ML**: Training models to learn patterns from data.
- **DL**: A subset of ML utilizing deep multi-layered neural networks (inspired by the human brain) to model complex patterns.

### Core Components of ML
1. **Dataset**: A collection of historical records (e.g. IPL match statistics or housing prices).
2. **Features**: Independent input variables (columns $X$) used by the model to learn patterns.
3. **Label / Target**: The dependent output variable ($y$) that we want to predict (e.g. who wins the match).
4. **Model**: A mathematical algorithm (like Linear Regression or a Decision Tree) trained on features to predict the target.
5. **Training**: The process where the model fits its parameters to find relationships in the data.
6. **Prediction**: The output generated by the trained model when given new feature values.

### Types of Machine Learning
1. **Supervised Learning**: The model is trained on **labeled data** (input + correct output are provided).
   - *Examples*: Spam detection, house price prediction, image classification.
2. **Unsupervised Learning**: The model is trained on **unlabeled data** to discover hidden groupings.
   - *Examples*: Customer segmentation, recommendation systems, anomaly detection.
   - *Algorithms*: K-Means Clustering, Principal Component Analysis (PCA).
3. **Reinforcement Learning**: An agent learns by taking actions in an environment to maximize cumulative rewards.
   - *Examples*: Robotics, game playing (AlphaGo), self-driving cars.

### Machine Learning Workflow
```text
Collect Data ──> Clean/Preprocess ──> Choose Model ──> Train Model ──> Evaluate ──> Deploy
```

## 2. Key Modeling Concepts

### Training vs. Testing Data
To evaluate how well an ML model performs, we split the historical dataset:
- **Training Data (~80%)**: Used to train the model so it learns the underlying patterns.
- **Testing Data (~20%)**: Held out during training and used purely to test the model's performance on unseen data.

```text
Dataset (100%)
  ├── Training Set (80%) ──> Model learns rules
  └── Testing Set (20%)  ──> Evaluates prediction performance
```

### Overfitting vs. Underfitting

- **Underfitting**: The model is too simple and fails to learn the patterns in the training data. (Low training accuracy, low test accuracy).
- **Overfitting**: The model is too complex and memorizes details/noise in the training data rather than generalizing. (High training accuracy, low test accuracy).

| Feature | Overfitting | Underfitting |
| :--- | :--- | :--- |
| **Learning state** | Learns too much / memorizes | Learns too little / fails to capture trend |
| **Training Accuracy** | Extremely High | Low |
| **Testing Accuracy** | Low | Low |
| **Cause** | Overly complex model, small dataset, too many epochs | Model too simple, insufficient training, missing features |
| **Analogy** | Memorizing exam questions exactly | Skimming only one page before the exam |

### Model Evaluation Metrics
Once a model is trained, its predictions are compared against actual values using metrics.

#### 1. Classification Metrics (Categorical Outputs)
- **Accuracy**: The fraction of total predictions the model got correct.
  
  $$\text{Accuracy} = \frac{\text{Correct Predictions}}{\text{Total Predictions}}$$

- **Precision**: Out of all the positive predictions made by the model, how many are actually positive?
  
  $$\text{Precision} = \frac{TP}{TP + FP}$$

- **Recall (Sensitivity)**: Out of all actual positive cases, how many did the model correctly identify?
  
  $$\text{Recall} = \frac{TP}{TP + FN}$$

- **F1-Score**: The harmonic mean of Precision and Recall, providing a balanced measure for imbalanced datasets.
  
  $$\text{F1-Score} = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}$$

##### Confusion Matrix
A tabular layout of classification results:

| Actual \ Predicted | Predicted YES | Predicted NO |
| :--- | :--- | :--- |
| **Actual YES** | **True Positive (TP)** | False Negative (FN) |
| **Actual NO** | False Positive (FP) | **True Negative (TN)** |

- **TP**: Actual positive correctly predicted as positive.
- **TN**: Actual negative correctly predicted as negative.
- **FP**: Actual negative incorrectly predicted as positive (Type I Error).
- **FN**: Actual positive incorrectly predicted as negative (Type II Error).

#### 2. Regression Metrics (Numerical Outputs)
- **Mean Squared Error (MSE)**: The average of the squared differences between actual and predicted values. A lower MSE indicates a better fit.
  
  $$\text{MSE} = \frac{1}{n} \sum_{i=1}^n (y_i - \hat{y}_i)^2$$

## 3. Supervised Learning: Regression vs. Classification

### Regression Overview
Regression is used to predict **continuous numerical values**.
- *Mathematical Line of Best Fit*: Linear Regression finds $y = mx + b$.
- *Examples*: Price, Temperature, Salary, Stock Value.
- *Algorithms*: Linear Regression, Decision Tree Regressor, Random Forest Regressor.

### Classification Overview
Classification is used to predict **categorical labels/classes**.
- *Decision Boundaries*: Splits data into separate classes.
- *Types*:
  - **Binary**: Two classes (e.g. Spam / Not Spam).
  - **Multi-Class**: More than two classes (e.g. Cat / Dog / Horse).
  - **Multi-Label**: A single record belongs to multiple classes (e.g. Movie Genres).
- *Algorithms*: Logistic Regression (predicts probabilities using a Sigmoid curve), Decision Tree Classifier, Random Forest Classifier, Support Vector Machine (SVM), K-Nearest Neighbors (KNN).

### Comparative Summary

| Feature | Regression | Classification |
| :--- | :--- | :--- |
| **Output Type** | Continuous Numerical | Categorical / Discrete Classes |
| **Goal** | Predict quantity / value | Assign labels to classes |
| **Example Output** | E.g. ₹55,000, 32.5 °C | E.g. "Spam", "Pass", "Approve" |
| **Main Algorithms** | Linear Regression, Random Forest Regressor | Logistic Regression, Decision Tree, SVM, KNN |

## 4. Case Study: IPL Dataset EDA

Exploratory Data Analysis was performed on the historical IPL matches dataset (`matches.csv`) to extract trends and player statistics.

### Data Cleaning & Preparation
- **Missing Value Identification**: Checked with `df.isnull().sum()`. Identified missing values in `city` and `player_of_match`.
- **City Column**: Missing values imputed using the mode:
  ```python
  df['city'] = df['city'].fillna(df['city'].mode()[0])
  ```
- **Player of Match Column**: Missing values filled with a default string:
  ```python
  df['player_of_match'] = df['player_of_match'].fillna('unknown')
  ```
- **Dropping Irrelevant Columns**: Dropped columns with excessive null values like `method`.
  ```python
  df.drop('method', axis=1, inplace=True)
  ```

### Analytical Insights & Visualizations
1. **Total Wins per Team**: Calculated using `.value_counts()` on the `winner` column and visualized via a horizontal bar chart:
   ```python
   win = df['winner'].value_counts()
   plt.barh(y=win.index, width=win.values, color='skyblue')
   plt.gca().invert_yaxis() # Invert y-axis to show the highest wins on top
   ```
2. **Match Outcome Distribution (Runs vs. Wickets)**: Visualized as a Pie Chart using `.value_counts()` on the `result` column, indicating that chasing (winning by wickets) is slightly more common than defending (winning by runs).
3. **Toss Winner vs. Match Winner**: Analyzed if winning the toss offers a significant advantage. The data shows that the toss winner wins the match only **~50.59%** of the time, suggesting that the toss decision does not guarantee a victory.
   ```python
   toss_and_match_winner = (df['toss_winner'] == df['winner']).sum()
   win_percentage = (toss_and_match_winner * 100) / len(df)
   print(f"Toss Winner = Match Winner: {win_percentage.round(2)}%")
   ```
4. **Toss Decision Choices**: Count of teams choosing to bat vs. field. Historically, winning by fielding first is preferred (700 wins choosing to field vs. 390 wins choosing to bat).
5. **Top Match Players**: Top 10 players winning the most "Player of the Match" awards (led by AB de Villiers with 25, followed by Chris Gayle with 22).

### Debugging & Dataset Resolution Notes

> [!WARNING]
> **KeyError: 'won_by' or 'best_bowling_figure'**
>
> In the `IPL_Capstone_Project.ipynb` notebook, you might encounter KeyErrors when attempting to use columns like `won_by`, `top_scorer`, `highscore`, or `best_bowling_figure`. 
>
> This happens because there are two different datasets:
> 1. **Historical IPL Matches (`matches.csv`)**: Contains data from 2008 to 2024. Its columns include `winner`, `result` (instead of `won_by`), and `result_margin` (instead of `margin`). It **does not** contain individual batting or bowling stats like `top_scorer` or `best_bowling`.
> 2. **IPL 2022 Match-level Dataset**: A separate file containing 2022 specific rows with columns for individual player match stats (`top_scorer`, `highscore`, `best_bowling`, `best_bowling_figure`).
>
> **How to resolve these errors:**
> - To check how teams won in the historical dataset, replace `won_by` with `result`:
>   ```python
>   # Correct syntax for matches.csv:
>   sns.countplot(x=df['result'])
>   plt.title("Won By Outcome (Wickets/Runs/Tie)")
>   ```
> - To run individual stats (`top_scorer`, `best_bowling_figure`), ensure you load the specific IPL 2022 dataset CSV instead of the historical `matches.csv` (which has only match-level outcome features).
